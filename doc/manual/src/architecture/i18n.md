# Internacionalización

SciNodes soporta múltiples idiomas con un sistema simple
key→string en JSON.

## API

```cpp
#include "i18n/Strings.hpp"

const std::string s = scinodes::tr("statusbar.run");
// o
const std::string s = scinodes::trOr("node.UnknownType.label", "Tipo desconocido");
```

| Función                  | Comportamiento si la key no existe                                |
|--------------------------|--------------------------------------------------------------------|
| `tr(key)`                | Devuelve la key misma + log warning.                              |
| `trOr(key, fallback)`    | Devuelve `fallback` (típicamente la versión inglesa hardcoded).   |

`trOr` es el patrón **preferido** para strings nuevos — permite
introducir un fallback antes de tener la traducción lista.

## Archivos

| Path                | Idioma            |
|---------------------|-------------------|
| `i18n/en.json`      | Inglés (canónico) |
| `i18n/es.json`      | Español           |

Ejemplo `es.json`:

```json
{
  "statusbar.run":            "▶ Ejecutar",
  "statusbar.pause":           "⏸ Pausar",
  "statusbar.state.empty":    "Vacío",

  "node.StepSignal.label":    "Paso",
  "node.StepSignal.tooltip":  "Genera un escalón de amplitud configurable",

  "find.header":              "Buscar nodo",
  "find.search_hint":         "Empezá a escribir...",
  "find.no_results":          "Sin resultados",
  "find.breadcrumb_root":     "Raíz"
}
```

## Convención de keys

| Patrón                                  | Uso                                       |
|-----------------------------------------|-------------------------------------------|
| `statusbar.<elemento>`                  | Strings de la barra de estado             |
| `panel.<panelName>`                     | Títulos de paneles                        |
| `node.<NodeType>.label`                 | Nombre legible del nodo                   |
| `node.<NodeType>.tooltip`               | Tooltip del nodo                          |
| `node.<NodeType>.input_label.<index>`   | Label de un input específico              |
| `node.<NodeType>.output_label.<index>`  | Label de un output específico             |
| `node.<NodeType>.param.<name>.label`    | Label de un parámetro                     |
| `find.*`, `error.*`, etc.               | Por categoría funcional                   |

## Estado

| Categoría                | Cobertura es.json |
|--------------------------|-------------------|
| StatusBar                | ✅ 100%           |
| Menús (Archivo, Editar)  | ✅ 100%           |
| NodeTypes (labels)       | ✅ 100% (etapa 6L) |
| Tooltips                 | ⚠ ~60%           |
| Diagnósticos de gramática| ⚠ ~40%           |

## Agregar un idioma

1. Copiar `i18n/en.json` a `i18n/<lang>.json`.
2. Traducir los valores (no las keys).
3. Agregar el idioma al dropdown en `View → Idioma`.
4. Recompilar.

No hay restricciones de encoding — el loader lee UTF-8.
Caracteres con diacríticos funcionan tal cual.

## Cómo se aplica el cambio

`scinodes::setLanguage("es")` recarga el JSON y dispara una
notificación que invalida los caches de strings de los
paneles.  El UI se redibuja en el siguiente frame con los
nuevos textos.  No requiere reiniciar.
